# CEC2005 Validation Data Set

This directory contains authoritative validation data for the 25 CEC2005 benchmark functions, generated by the modernized CEC2005-C codebase.

## Overview
- **Functions:** 25 (f01–f25), as defined in the [CEC2005 Tech Report](../CEC2005_Tech-Report.md)
- **Dimensions:** 2, 10, 30, 50
- **Inputs per function/dimension:**
  - `optimum`: The true global optimum (shift vector or equivalent, extracted from the C code output)
  - `random_input`: A random vector within the search space
  - `lower`: All variables at lower bound
  - `upper`: All variables at upper bound
- **Output:**
  - One JSON file per function: `f<id>.json`
  - Each file contains results for all supported dimensions as an array of cases
  - Each file includes the function number, official name, bounds, and bias

## JSON Schema
Each file follows this structure:

```
{
  "function": <int>,           // CEC2005 function number (1–25)
  "name": <str>,               // Official function name from the CEC2005 Tech Report
  "bias": <float>,             // Bias for this function
  "cases": [
    {
      "dimension": <int>,          // Problem dimension (2, 10, 30, 50)
      "initialization_bounds": {
        "low": [<float>, ...],     // Lower bounds for each dimension (array length = dimension)
        "high": [<float>, ...]     // Upper bounds for each dimension (array length = dimension)
      },
      "operational_bounds": {
        "low": [<float>, ...],     // Lower bounds for each dimension (array length = dimension)
        "high": [<float>, ...]     // Upper bounds for each dimension (array length = dimension)
      },
      "optimum": [<float>, ...],   // Global optimum vector (from C output)
      "random_input": [<float>, ...], // Random vector within bounds
      "outputs": {
        "optimum": <float>,        // Objective value at optimum
        "random": <float>,         // Objective value at random_input
        "lower": <float>,          // Objective value at lower bound
        "upper": <float>           // Objective value at upper bound
      }
    },
    ...
  ]
}
```

- All numeric values are full-precision floats.
- The `optimum` vector and `bias` are directly parsed from the CEC2005-C application output for each function/dimension.
- The `name` field is the official function name from the CEC2005 Tech Report.
- The bounds for each case are explicit and per-dimension, provided as `initialization_bounds` and `operational_bounds` objects within each case.

## Usage
- **Regression Testing:**
  - Use the input vectors and expected outputs to verify correctness of new implementations.
  - The `optimum` input and output provide a ground truth for each function/dimension.
- **Benchmarking:**
  - Compare algorithm results against these values for reproducibility and correctness.
- **Reproducibility:**
  - All data is generated from the actual CEC2005-C code, using the official shift vectors and biases.

## Authority
- **Source:** All data is generated using the CEC2005-C codebase, with bounds and function definitions taken from the [CEC2005 Tech Report](../CEC2005_Tech-Report.md).
- **Optimum and Bias:** The optimum vector and bias are extracted from the C code output for each function/dimension, ensuring full fidelity to the reference implementation.
